📋 시간표 — 주요 기능

🎯 학생 통계 배지 (상단 헤더)

  • 재원: 현재 등록 학생 (녹색)
  • 신입: 30일 이내 등록 (핑크)
  • 예정: 대기 + 퇴원예정 (앰버)
  • 퇴원: 30일 이내 퇴원 (회색)
  • 배지 클릭 시 드롭다운으로 상세 목록

🖱️ 드래그 & 드롭

  • 학생 드래그로 반 이동
  • 이전 위치: 취소선 + 회색
  • 이동 위치: 보라색 배경
  • 저장 시 Firestore enrollment 분리 처리
  • Ctrl+Z로 이동 취소 가능

⌨️ 엑셀 모드 단축키

  • 클릭/Shift+클릭: 학생 다중 선택
  • Ctrl+C/X/V: 복사/잘라내기/붙여넣기
  • Del: 선택 학생 보류 삭제
  • Ctrl+Z: 마지막 작업 취소
  • Tab/Enter: 셀 이동
  • 자동완성 하이라이트 (다른 반 학생 빨간색)

🔍 필터 시스템

  • 강의실: 본원/바른/고등
  • 학교별: 초/중/고 (학교명)
  • 학년별: 초1~고3
  • 셔틀탑승: 메이크에듀 연동
  • 지각/결석: 주간 리포트 기반
  • 병합 셀 (합반) 지원

❓ 자주 묻는 질문

Q. dev 사이트와 prod 사이트의 차이?

📌 같은 Firestore DB를 공유합니다. 프론트엔드 코드만 분리된 테스트/운영 버전입니다. dev에서 테스트한 내용이 DB에 반영됩니다.

Q. 시뮬레이션 데이터가 실제에 영향?

📌 실제 반영 버튼을 누르지 않는 한 안전합니다. 메모리 내 편집이며, 시나리오 저장 시에만 math_scenarios 컬렉션에 기록됩니다.

Q. Master 권한 계정은?

📌 settings/system_config의 masterEmails에 등록된 계정이 Master입니다. 다른 역할로 시뮬레이션 테스트 가능.

Q. 메이크에듀 크롤링이 실패하면?

📌 로그인 차단(NO_REJECT) 또는 페이지 구조 변경이 원인일 수 있습니다. Cloud Function 로그를 확인하고 Secret Manager의 인증 정보를 점검하세요.

Q. 새 탭/기능 추가 방법?

📌 types/system.ts의 AppTab에 추가 → TAB_META/TAB_GROUPS 등록 → Layout/TabContent.tsx에 라우팅 → 권한 설정.

Q. 데이터 백업/복구?

📌 시뮬레이션 "실제 반영" 시 자동 백업이 math_scenarios/backup_*에 저장됩니다. 필요 시 로드하여 복구 가능합니다.
