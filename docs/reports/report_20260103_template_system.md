# 간트 차트 템플릿 시스템 구현 보고서

**날짜**: 2026년 01월 03일
**작성자**: Antigravity

## 1. 개요
사용자 요청에 따라 간트 차트의 프로젝트를 템플릿으로 저장하고, 저장된 템플릿을 불러와 새 프로젝트를 생성할 수 있는 기능을 구현했습니다. 또한 UI 구조를 개선하여 프로젝트와 템플릿을 명확히 구분했습니다.

## 2. 주요 변경 사항

### 2.1 데이터 모델 확장 (`types.ts`)
- `GanttTemplate` 인터페이스에 `isTemplate?: boolean` 필드를 추가하여 일반 프로젝트와 재사용 가능한 템플릿을 구분했습니다.

### 2.2 템플릿 저장 기능 (`GanttChart.tsx`, `GanttManager.tsx`)
- **기능**: 현재 보고 있는 프로젝트를 "템플릿으로 저장"할 수 있습니다.
- **UI**: 간트 차트 상단 헤더에 `템플릿으로 저장` 버튼을 추가했습니다.
- **로직**:
  - 버튼 클릭 시 템플릿 이름을 입력받습니다.
  - 현재 프로젝트의 모든 데이터(태스크, 의존성 등)를 복제하여 `isTemplate: true`로 설정 후 저장합니다.
  - ID는 Firestore에서 새로 생성되도록 처리하여 기존 프로젝트와 충돌을 방지했습니다.

### 2.3 템플릿 불러오기 기능 (`GanttBuilder.tsx`)
- **기능**: 새 프로젝트 생성 시, 기존에 저장된 템플릿을 선택하여 내용을 자동으로 채울 수 있습니다.
- **UI**: '새 프로젝트 생성' 화면의 좌측 사이드바 상단에 "템플릿에서 불러오기" 드롭다운 메뉴를 추가했습니다.
- **로직**:
  - 템플릿 선택 시 해당 템플릿의 태스크 목록을 깊은 복사(Deep Copy)하여 현재 작성 중인 폼에 반영합니다.
  - 기존 입력 데이터가 있는 경우 덮어쓰기 경고를 표시합니다.

### 2.4 UI 및 UX 개선 (`GanttManager.tsx`)
- **사이드바 분리**: 기존의 단일 리스트를 '프로젝트 바로가기'와 '템플릿 라이브러리' 두 개의 섹션으로 분리하여 가독성을 높였습니다.
- **자동 뷰 리셋**: 프로젝트 삭제 시, 삭제된 프로젝트를 보고 있었다면 자동으로 홈 화면으로 이동하도록 수정하여 화면에 잔상이 남는 버그를 해결했습니다.
- **오류 수정**: `GanttManager.tsx` 내의 불필요한 중첩 태그 및 문법 오류를 수정하여 안정성을 확보했습니다.

## 3. 결과
- 사용자는 반복되는 프로젝트 구조를 매번 새로 입력할 필요 없이, 템플릿으로 저장해두고 원클릭으로 불러와 사용할 수 있게 되었습니다.
- 프로젝트 목록과 템플릿 목록이 분리되어 관리가 용이해졌습니다.
