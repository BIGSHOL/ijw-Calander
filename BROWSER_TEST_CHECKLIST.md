## 1. Z-index 레이어 테스트

### 1.1 기본 레이어 순서 확인

- [ ] **네비게이션 (z-30)**
  - [ ] 네비게이션 바가 메인 컨텐츠 위에 표시됨
  - [ ] 탭 전환 시 네비게이션이 항상 보임
  - [ ] 스크롤 시 고정 위치 유지

- [ ] **일반 모달 (z-[100])**
  - [ ] 모달이 네비게이션 위에 표시됨
  - [ ] 모달 배경 오버레이가 전체 화면을 덮음
  - [ ] 모달 외부 클릭 시 닫힘

- [ ] **서브 모달 (z-[110])**
  - [ ] 서브 모달이 일반 모달 위에 표시됨
  - [ ] 예: 시간표 시나리오 관리 → 저장 확인 다이얼로그
  - [ ] 서브 모달 닫기 시 일반 모달이 그대로 남아있음

- [ ] **최상위 요소 (z-[120])**
  - [ ] MemoDropdown이 모든 모달 위에 표시됨
  - [ ] 툴팁이 모든 요소 위에 표시됨

### 1.2 레이어 충돌 확인

- [ ] **모달 + 드롭다운**
  - [ ] 모달을 연 상태에서 헤더의 드롭다운 메뉴가 정상 작동
  - [ ] 드롭다운이 모달 위에 표시되지 않음 (정상)

- [ ] **모달 + MemoDropdown**
  - [ ] 모달을 연 상태에서 MemoDropdown 클릭
  - [ ] MemoDropdown이 모달 위에 표시됨 (z-[120] > z-[100])

- [ ] **서브 모달 + MemoDropdown**
  - [ ] 서브 모달을 연 상태에서 MemoDropdown 클릭
  - [ ] MemoDropdown이 서브 모달 위에 표시됨 (z-[120] > z-[110])

### 1.3 특정 시나리오 테스트

- [ ] **시간표 → 시나리오 관리 → 저장 확인**
  1. 시간표 탭 이동
  2. 시나리오 관리 모달 열기 (z-[100])
  3. 저장 버튼 클릭 → 확인 다이얼로그 (z-[110])
  4. 확인 다이얼로그가 시나리오 관리 모달 위에 표시되는지 확인

- [ ] **학생 상세 → 설정 모달**
  1. 학생 목록에서 학생 클릭 → 상세 모달 (z-[100])
  2. 설정 버튼 클릭 → 설정 모달 (z-[100])
  3. 두 모달이 겹쳐 보이지 않고 전환되는지 확인

- [ ] **출석부 → 학생 추가 → MemoDropdown**
  1. 출석부 탭 이동
  2. 학생 추가 모달 열기 (z-[100])
  3. 헤더의 MemoDropdown 클릭
  4. MemoDropdown이 모달 위에 표시되는지 확인

## 2. AddStudentToAttendanceModal 기능 테스트

### 2.1 모달 열기/닫기

- [ ] **모달 열기**
  - [ ] 출석부 탭 → "학생 추가" 버튼 클릭
  - [ ] 모달이 부드럽게 나타남
  - [ ] 모달이 화면 중앙에 위치

- [ ] **모달 닫기**
  - [ ] 우측 상단 X 버튼으로 닫기
  - [ ] 모달 외부 클릭으로 닫기
  - [ ] "취소" 버튼으로 닫기
  - [ ] ESC 키로 닫기 (만약 구현되어 있다면)

### 2.2 섹션별 학생 분류

- [ ] **현재 출석부 섹션**
  - [ ] 이미 출석부에 있는 학생이 "현재 출석부" 섹션에 표시됨
  - [ ] 학생 카드가 회색으로 표시됨 (disabled 상태)
  - [ ] "이미 추가됨" 라벨이 표시됨
  - [ ] 클릭 시 선택되지 않음

- [ ] **담임 학생 섹션**
  - [ ] 현재 선생님의 수업에 등록된 학생이 표시됨
  - [ ] 섹션 헤더가 파란색 (text-indigo-600)
  - [ ] 학생 수가 정확하게 표시됨 (예: "담임 학생 (5명)")
  - [ ] 클릭 시 선택/해제 가능

- [ ] **검색 결과 섹션**
  - [ ] 검색어 입력 전에는 표시되지 않음
  - [ ] 검색어 입력 후 매칭되는 학생만 표시됨
  - [ ] 섹션 헤더가 "검색 결과 (N명)"으로 표시됨
  - [ ] 클릭 시 선택/해제 가능

### 2.3 검색 기능

- [ ] **기본 검색**
  - [ ] 학생 이름으로 검색 (예: "이소은")
  - [ ] 영어 이름으로 검색 (예: "Alice")
  - [ ] 학교로 검색 (예: "대구")

- [ ] **검색 결과**
  - [ ] 검색어와 일치하는 학생만 표시됨
  - [ ] 부분 일치 검색 가능 (예: "대" → "대구평광중", "대명초")
  - [ ] 대소문자 구분 없이 검색됨
  - [ ] 검색어 지우면 원래 상태로 돌아감

- [ ] **빈 검색 결과**
  - [ ] 검색 결과가 없을 때 "검색 결과가 없습니다." 메시지 표시
  - [ ] 검색어가 없고 담임 학생도 없을 때 안내 메시지 표시

### 2.4 학생 선택/해제

- [ ] **단일 선택**
  - [ ] 학생 카드 클릭 시 선택됨
  - [ ] 체크박스에 체크 표시
  - [ ] 카드 배경이 파란색으로 변경 (bg-indigo-50)
  - [ ] 다시 클릭 시 선택 해제

- [ ] **다중 선택**
  - [ ] 여러 학생을 동시에 선택 가능
  - [ ] 선택된 학생 수가 푸터에 표시됨 (예: "3명 선택됨")
  - [ ] 버튼 텍스트가 업데이트됨 (예: "3명 추가")

- [ ] **선택 불가 학생**
  - [ ] "현재 출석부" 섹션의 학생은 선택 불가
  - [ ] 클릭 시 아무 반응 없음

### 2.5 등록 타입 변경

- [ ] **특강/보강 (temporary)**
  - [ ] 기본값이 "특강/보강"으로 선택됨
  - [ ] 버튼이 노란색으로 강조됨 (bg-amber-100)
  - [ ] 아이콘이 표시됨 (Target)

- [ ] **정규 등록 (regular)**
  - [ ] 클릭 시 "정규 등록"으로 변경
  - [ ] 버튼이 파란색으로 강조됨 (bg-indigo-100)
  - [ ] 아이콘이 표시됨 (BookOpen)

### 2.6 학생 추가

- [ ] **학생 추가 버튼**
  - [ ] 학생 선택 전: 버튼 비활성화 (회색)
  - [ ] 학생 선택 후: 버튼 활성화 (파란색)
  - [ ] 버튼 텍스트가 선택된 학생 수를 표시 (예: "3명 추가")

- [ ] **추가 프로세스**
  - [ ] "추가" 버튼 클릭
  - [ ] 버튼 텍스트가 "추가 중..."으로 변경
  - [ ] 로딩 중 버튼 비활성화
  - [ ] 추가 완료 후 모달 자동 닫힘

- [ ] **데이터 반영**
  - [ ] 출석부에 추가된 학생이 표시됨
  - [ ] 추가된 학생의 등록 타입이 정확함 (특강/보강 or 정규)
  - [ ] 캐시가 즉시 업데이트됨 (새로고침 불필요)

### 2.7 에러 처리

- [ ] **네트워크 에러**
  - [ ] 네트워크 끊김 시 에러 메시지 표시
  - [ ] 알림: "학생 추가 중 오류가 발생했습니다."
  - [ ] 모달이 닫히지 않고 재시도 가능

- [ ] **권한 에러**
  - [ ] 권한이 없을 경우 적절한 에러 메시지
  - [ ] (권한 관련 로직이 있다면)
